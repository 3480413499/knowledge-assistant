# 知识小助手 (Knowledge Assistant)

> RAG 智能知识库问答系统 — 上传文档，自然语言提问，AI 精准回答并标注来源

## 项目背景

- 华东理工大学信息工程专业大三本科生的实习项目
- 目标：2 周内完成，作为暑期实习 & 秋招的展示项目
- 2026 年 AI 实习生最热门方向：RAG（检索增强生成）应用

## 整体架构

```
┌──────────────────────────────────────┐
│         🖥️ 前端 (Streamlit)          │  ← 用户界面：上传文件 + 提问
├──────────────────────────────────────┤
│         ⚙️ 业务逻辑层                  │  ← 处理请求，协调各组件
├──────────────────────────────────────┤
│  📦 ChromaDB         │   🤖 LLM API  │  ← 数据层：向量检索 + AI生成
│  + BGE-M3 Embedding  │  (DeepSeek)   │
└──────────────────────────────────────┘
```

### 工作流程

1. 用户上传 PDF → 提取文字 → 分块 → BGE-M3 转向量 → 存入 ChromaDB
2. 用户提问 → 问题转向量 → ChromaDB 搜索相关片段 → 片段 + 问题 → LLM 生成回答 + 标注来源

## 技术选型

| 层 | 选型 | 理由 |
|---|------|------|
| 🖥️ 前端 | Streamlit | Python 写界面，零前端学习成本 |
| ⚙️ 后端 | FastAPI（内嵌 Streamlit） | 轻量异步框架，面试常考 |
| 🧠 LLM | DeepSeek API | 便宜、中文强、API 兼容 OpenAI |
| 📦 向量数据库 | ChromaDB | 轻量级，pip install 即用，无需单独部署 |
| 🔢 嵌入模型 | BGE-M3（本地） | 免费，C-MTEB 中文榜单前列，面试加分项 |
| 📄 PDF 解析 | PyMuPDF | 中文支持好，已预装 |

### Embedding 模型对比

| | BGE-M3（本地） | DeepSeek Embedding（API） |
|---|---|---|
| 💰 费用 | 免费 | 按 token 收费 |
| 🇨🇳 中文能力 | C-MTEB 前列 | 强 |
| 🚀 速度 | 看硬件 | 稳定 |
| 🎤 面试价值 | 高（本地部署，理解原理） | 中（API Caller 印象） |

**选择 BGE-M3**：免费 + 开源本地部署在面试中是加分项。

## 功能设计

### 知识库管理页（侧边栏）
- 📄 文件上传区（拖拽或点击，支持 PDF/Word/TXT）
- 📋 已上传文档列表，显示处理状态
- 🗑️ 一键清空知识库

### 问答对话页（主体）
- 💬 对话式聊天界面
- 📎 每条回答下方折叠显示参考来源（文件 + 页码 + 原文片段）
- 🔄 可切换 LLM 模型

### 回答格式

```
AI回答内容...
─────────────────
📎 参考来源：
[1] 《AR感知综述.pdf》 第3页 — 「视觉SLAM是一种...」
[2] 《课程笔记.docx》 第1页 — 「SLAM即Simultaneous...」
```

## 文档处理流水线

```
PDF上传 → PyMuPDF提取文字 → 智能分块(500字/块+50字重叠) → BGE-M3转向量 → 存入ChromaDB
```

- **分块策略**：段落级 + 500 字/块 + 50 字重叠（防止语义截断）
- **去重**：MD5 指纹检测，同一文件自动跳过

## 错误处理 & 边界情况

| 场景 | 处理方式 |
|------|---------|
| 📄 文件过大（>50MB） | 拦截并提示拆分 |
| 🔒 PDF 为扫描图片 | 检测文字量为 0 时提示先 OCR |
| ❓ 问题与知识库无关 | 提示"知识库暂无相关内容" |
| 🐌 处理耗时 | 进度条 + 预估时间 |
| 🌐 API 失败 | 自动重试 2 次，失败提示检查 Key |
| 🔁 重复文件 | MD5 检测，提示跳过 |
| 📝 空知识库提问 | 提示先上传文档 |
